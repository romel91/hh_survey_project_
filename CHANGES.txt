================================================================
NEW FILES CREATED
================================================================

1. scripts/config.py
   ─────────────────
   Purpose : Single shared configuration file for all scripts.
   Why     : All three scripts previously duplicated the same
             file path strings and POVERTY_LINE value. A change
             to any path or threshold required editing all three
             files. Now changed in one place.
   Contents:
     - RAW_PATH     = 'data/raw/HH_Survey_Dataset.xlsx'
     - CLEANED_PATH = 'data/cleaned/HH_Survey_Cleaned.csv'
     - HH_PATH      = 'data/cleaned/HH_Household_Level.csv'
     - CHARTS_DIR   = 'charts'
     - OUTPUT_PATH  = 'outputs/HH_Survey_Analysis.xlsx'
     - POVERTY_LINE = 5000  (BDT per capita per month)

2. run_all.py
   ──────────
   Purpose : Single entry point to run the full pipeline.
   Why     : Previously the three scripts had to be run manually
             in the correct order. run_all.py runs them in
             sequence (01 → 02 → 03), reports elapsed time per
             script, and stops immediately with an error message
             if any script fails.
   Usage   : python3 run_all.py  (from project root)

3. requirements.txt
   ─────────────────
   Purpose : Lists all Python package dependencies.
   Why     : The project had a venv/ folder but no requirements
             file, making the environment impossible to reproduce
             without running the code and seeing what imports.
   Contents: pandas, numpy, openpyxl, matplotlib, seaborn, scipy


================================================================
CHANGES TO scripts/01_data_cleaning.py
================================================================

1. Replaced local path constants with config import
   BEFORE:
     RAW_PATH     = 'data/raw/HH_Survey_Dataset.xlsx'
     CLEANED_PATH = 'data/cleaned/HH_Survey_Cleaned.csv'
     HH_PATH      = 'data/cleaned/HH_Household_Level.csv'
   AFTER:
     from config import RAW_PATH, CLEANED_PATH, HH_PATH, POVERTY_LINE

2. Added income_missing flag before filling NaN income with 0
   BEFORE:
     df['monthly_income'] = pd.to_numeric(...).fillna(0)
   AFTER:
     df['income_missing'] = df['monthly_income'].isnull().astype(int)
     df['monthly_income'] = pd.to_numeric(...).fillna(0)
   Why: Filling NaN with 0 silently conflated "no income recorded"
        with "earns zero". The income_missing=1 flag preserves
        the distinction so analysts can filter on it separately
        (e.g. exclude imputed zeros from earner-only analyses).

3. Fixed interview_duration household aggregation
   BEFORE:
     interview_duration = ('interview_duration', 'first')
   AFTER:
     interview_duration = ('interview_duration', 'mean')
   Why: 'first' just copied the Head row's interview duration to
        the household record. Since interview_duration is measured
        per individual, 'mean' gives a more meaningful
        household-level summary.

4. Removed local POVERTY_LINE = 5000 constant
   BEFORE:
     POVERTY_LINE = 5000
     hh['is_poor'] = (hh['income_per_capita'] < POVERTY_LINE)...
   AFTER:
     hh['is_poor'] = (hh['income_per_capita'] < POVERTY_LINE)...
     (POVERTY_LINE now imported from config.py)
   Why: Eliminates the risk of updating the threshold in one script
        but forgetting to update it in another.
